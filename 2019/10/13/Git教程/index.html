Git基本操作

  • git init —— 初始化仓库

  • git status —— 查看仓库状态

  • git add —— 向暂存区中添加文件

  • git commit —— 保存仓库的历史记录

    1. git commit -m ‘message’ —— 记录一行提交信息

    2. git commit —— 记录相信提交的信息(git需要配置默认的编辑器，格式按照下面)。如果在编辑器启动后想中止提交，请将信息留空并关闭编辑器。

      • 第一行：用一行文字简述提交的内容
      • 第二行：空行
      • 第三行以后：记述更改的原因和详细内容

    3. git commit —— 查看提交后的状态

  • git commit -am ‘message’ —— 一次完成git add与git commit命令

  • 在相关Issue中显示提交

  • 关闭issue

  • git log —— 查看提交日志

    1. git log –pretty=short —— 只显示提交信息的第一行
    2. git log –pretty=oneline —— 提交信息在一行显示
    3. git log file/dir —— 只显示指定目录、文件的日志
    4. git log -p file —— 只显示文件的改动

  • git diff —— 查看更改前后的差别

    1. git diff head —— 查看工作树和最新提交的差别
    2. 在执行git commit命令之前先执行git diff head命令，查看本次提交与上次提交之间有什么差别。head指向当前分支中最新一次提交的指针。

分支的操作

  • git branch —— 显示分支列表

  • git checkout -b branck-name —— 创建、切换分支 该条命令等价于以下两条命令

    特性分支是指集中实现单一特性，除此之外不进行任务作业的分支

  • git checkout master —— 切换到master分支

  • git merge —— 合并分支

  • git log -‌-graph —— 以图表形式查看分支

  • git branch -D branch-name —— 删除本地分支

  • git branch -a —— 查看当前分支的相关信息

更改提交的操作

  • git reset —— 回溯历史版本

  • git reflog —— 查看当前仓库执行过的操作日志

  • 消除合并冲突

  • git commit –amend —— 修改提交信息

  • git rebase -i —— 压缩历史记录

    如果发现已提交的内容中有些许拼写错误等，可以提交一个修改，然后将这个修改包含到前一个提交之中，压缩成一个历史纪录。

推送至远程仓库

在GitHub上创建一个新的仓库，名字与本地仓库保持一致。创建时不要勾选Initialize this repository with a README选项。

  • git remote add —— 添加远程仓库

  • git push —— 推送至远程仓库master分支

从远程仓库获取

  • git clone —— 获取远程仓库(master分支)

  • 获取远程的其他分支

获取最新的远程仓库分支

  • git pull —— 获取最新的远程仓库分支
